## 🎤 面试加分项（必须准备）

### 原理深度问答

**Q1: Vue 3 的响应式原理是什么？**
> 答：Vue 3 使用 Proxy 对象代理数据，拦截 get/set/deleteProperty 等操作。get 时进行依赖收集（track），set 时触发更新（trigger）。相比 Vue 2 的 `Object.defineProperty`，Proxy 可以监听数组变化和新增属性，且性能更好。

**Q2: 为什么推荐使用 `ref` 而不是 `reactive`？**
> 答：`ref` 可以处理任何类型，且解构不会丢失响应性；`reactive` 只对对象有效，解构后响应性丢失。推荐统一使用 `ref` + `toRefs` 模式。

**Q3: `nextTick` 的原理？**
> 答：`nextTick` 利用事件循环的微任务机制（优先使用 `Promise.then`，降级使用 `MutationObserver` 或 `setTimeout`），在 DOM 更新完成后执行回调，确保你能获取到最新 DOM 状态。

**Q4: diff 算法的优化策略？**
> 答：Vue 采用**同层比较**策略，不跨层级对比。通过 `key` 值进行节点复用，使用**双端比较**算法（头头、尾尾、头尾、尾头）提高对比效率。在 Vue 3 中，还加入了**静态标记**（PatchFlags）来跳过静态节点的对比。
